= Ammitto Switzerland Sanctions Data Repository

== Purpose

This repository stores downloaded and processed sanctions data from the
Swiss State Secretariat for Economic Affairs (SECO) Sanctions Database.

The data is maintained by the Ammitto project and used by the `ammitto` Ruby
gem for querying international sanctions information.

== Legal Basis

Swiss sanctions are implemented through:

* The Federal Act on the Implementation of International Sanctions (Embargo Act, EmbA)
* The Ordinance on Measures against Specific Persons, Undertakings and Objects
* Various country-specific ordinances

Switzerland implements UN Security Council sanctions and also participates
in autonomous sanctions aligned with EU measures where consistent with
Swiss neutrality policy.

== Data Source

* **Authority**: State Secretariat for Economic Affairs (SECO) / Staatssekretariat für Wirtschaft
* **API Endpoint**: https://www.seco.admin.ch/seco/de/home/Aussenwirtschaftspolitik_Wirtschaftliche_Zusammenarbeit/Wirtschaftsbeziehungen/exportkontrollen-und-sanktionen/sanktionsmassnahmen/sanktionsliste.html
* **Format**: XML
* **Update Frequency**: As changes occur in Swiss sanctions regimes
* **Official Source**: https://www.seco.admin.ch/dam/seco/de/dokumente/Aussenwirtschaft/Wirtschaftsbeziehungen/exportkontrollen-sanktionen/Sanktionsliste/Sanktionsliste%20SECO.xml

== Directory Structure

```
.
├── downloaded/           # Raw data downloaded from SECO
│   └── ch_sanctions_list.xml
├── processed/            # Individual YAML files per sanction entity
│   └── {name}-{index}.yaml
├── reference-docs/       # Reference documentation
├── .github/workflows/    # GitHub Actions for automated updates
├── README.adoc           # This file
├── CLAUDE.md             # Guidance for Claude Code
└── update.log            # Timestamps of data updates
```

== Directory Descriptions

downloaded/:: Contains raw XML data files as downloaded from SECO.
The primary file is `ch_sanctions_list.xml` which contains the complete
sanctions database.

processed/:: Contains individual YAML files, one per sanction entity.
Files are named using the pattern `{primary-name}-{index}.yaml` where
index ensures uniqueness for entities with similar names.

reference-docs/:: Contains reference documentation about the Swiss
sanctions schema and data formats.

update.log:: Log file tracking when data was last updated with timestamps.

== Updating Data

Data is automatically updated daily via GitHub Actions.

To manually update using the ammitto gem:

[source,shell]
----
# From the ammitto gem directory
bundle exec ruby exe/ammitto fetch ch --format yaml --output-dir ./processed
----

Or via the adapters repository:

[source,shell]
----
# From the adapters/ directory
ruby adapter.rb
----

== YAML Schema

Each processed YAML file follows the schema defined in
`ammitto/lib/ammitto/sources/ch/` using Lutaml::Model.

.Example YAML structure
[source,yaml]
----
names:
  - "PRIMARY NAME"
  - "ALIAS 1"
entity_type: person  # or "organization"
source: ch
country: Country Name
birthdate: "YYYY-MM-DD"
identifications:
  - type: "Passport"
    number: "XXX123456"
addresses:
  - street: "Street Address"
    city: "City"
    country: "Country"
----

== Entity Types

The Swiss sanctions list contains the following entity types:

Person:: Individual persons subject to sanctions
Organization:: Legal entities, companies, and organizations
